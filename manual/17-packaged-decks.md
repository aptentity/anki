Anki 包（`.apkg` 文件）让你可以从其他用户导入牌组、笔记、笔记类型和卡片。通常在 [AnkiWeb](https://ankiweb.net/shared/decks) 上共享。

## 调度信息

Anki 包可能包含调度信息，这在你希望在不同设备或 profile 之间转移牌组时很有用。但是，当导入别人分享的牌组时，你通常**不希望继承他们的卡片间隔或复习历史**。

如果遇到导入卡片后间隔异常大，可能是牌组作者不小心包含了调度信息。你可以用浏览器的[设置到期日功能](https://docs.ankiweb.net/browsing.html#cards)来重置导入的卡片。在 Anki 23.10 及更新版本中，还可以在导入过程中**取消勾选**"Import any learning progress"（导入任何学习进度），直接不导入调度信息。这也会同时移除导入卡片上的"记忆蛀虫"或"已标记"标签。

## 更新

导入 `.apkg` 文件时，Anki 会识别其中与你之前导入过的笔记重复的笔记。默认情况下：如果文件中的笔记比本地版本更新，会用文件内容**更新**现有笔记。

如果笔记类型被修改过（例如你或牌组作者对笔记类型做了添加字段之类的改动），更新通常**无法进行**。你仍然可以导入文件中缺失的笔记，但如果牌组作者做了修改，之前导入的笔记不会被更新。

### Anki 23.10 及更新版本

Anki 23.10 引入了更多灵活性：

- 你可以选择**无条件更新**笔记和笔记类型，始终覆盖你的修改
- 或者选择**从不更新**现有对象

另外，如果你和牌组作者同时修改了同一个笔记类型，现在可以选择**合并**两个版本。这会保留两个版本中包含的所有模板和字段，但需要一次完整同步，且可能把其他现有笔记标记为已修改。

#### 给牌组作者的说明

合并功能依赖于模板和字段的 ID，这些 ID 是 Anki 2.1.67 引入的。如果某个模板或字段缺少 ID（因为它是用更早版本创建的），Anki 会尝试通过比较名称来找到对应项。

参见[此插件](https://ankiweb.net/shared/info/2063785767)，了解为什么**使用带 ID 的字段和模板来分享笔记类型更有优势**，以及如何给现有笔记类型添加 ID。
