## 反向卡片

你可以在 YouTube 上观看[关于反向卡片的视频](http://www.youtube.com/watch?v=DnbKwHEQ1mA&yt:cc=on)。

如果你想创建双向的卡片（例如"ookii"→"big" 和 "big"→"ookii"），你有几种选择。最简单的是选择内置的 **"Basic (and reversed card)"（基本-含反向卡）** 笔记类型。这会自动生成两张卡片，一个方向一张。

如果你只想为**部分**内容生成反向卡（也许你只想把时间花在最重要的内容上，或者有些卡片反过来没什么意义），可以选择 **"Basic (optional reversed card)"（基本-可选反向卡）** 笔记类型。这种笔记类型在你只填写前两个字段时只生成**正向**一张卡；如果你额外在"Add Reverse"字段里填点东西（比如一个"y"），Anki 就会再生成一张反向卡。**该字段的内容不会显示在卡片上**。

## 卡片的生成与删除

Anki 不会创建正面为空的卡片。因此，如果"My Field"为空，并且某张卡片的正面模板只包含该字段，那么这张卡片**不会被创建**。

当你编辑一条已添加的笔记时，如果原本为空的字段现在有了内容，Anki 会**自动创建额外的卡片**。但是，如果你的编辑让原本非空的字段变空了，Anki **不会立刻删除这些卡片**，因为那可能导致意外的数据丢失。要删除空卡片，请到主窗口中点击 **工具 → 空卡片（Empty Cards）**。系统会显示一张空卡片列表，供你选择删除。

由于卡片生成机制的原理，**无法手动删除单张卡片**——因为下次编辑笔记时它会被重新创建。正确做法是：把相关条件替换字段清空，然后用"空卡片"功能统一处理。

Anki 在判断卡片是否生成时**不考虑特殊字段或非字段文本**。因此，如果你的正面模板长这样：

```
Where is {{Country}} on the map?
```

当 Country 为空时，**不会生成卡片**。

## 选择性生成卡片

有时你可能只想为部分内容生成额外卡片，比如检验你对某组最重要单词的回忆能力。可以通过给笔记增加一个额外字段，并在希望生成额外卡的笔记中填入一些内容（如"1"）来实现。然后在卡片模板中，让卡片的生成依赖该字段非空。更多内容请看下面的[条件替换](#条件替换)章节。

## 条件替换

你可以让某些文本、字段或 HTML **只在某字段为空或非空时才出现**。示例：

```
这段文本始终显示。

{{#FieldName}}
 只有当 FieldName 有内容时，才显示这段文本
{{/FieldName}}

{{^FieldName}}
 只有当 FieldName 为空时，才显示这段文本
{{/FieldName}}
```

实际例子：仅当字段非空时显示一个标签：

```
{{#Tags}}
 标签：{{Tags}}
{{/Tags}}
```

或者假设你想在卡片正面以蓝色显示某个字段——前提是背面有额外的注释（也许"有注释"这件事本身能提醒你多花时间思考答案）。可以这样写：

```
{{#Notes}}
 <span style="color:blue;">
{{/Notes}}

{{FieldToFormat}}

{{#Notes}}
 </span>
{{/Notes}}
```

你也可以用条件替换**控制哪些卡片会被生成**。这是因为 Anki 不会生成正面为空的卡片。例如，假设一张卡片正面有两个字段：

```
{{Expression}}
{{Notes}}
```

正常情况下，只要 Expression 或 Notes 任一字段有内容，就会生成卡片。如果你**只想在 Expression 非空时生成卡片**，可以把模板改成：

```
{{#Expression}}
 {{Expression}}
 {{Notes}}
{{/Expression}}
```

如果你想**两个字段都必须有内容**，可以用嵌套的条件替换：

```
{{#Expression}}
 {{#Notes}}
 {{Expression}}
 {{Notes}}
 {{/Notes}}
{{/Expression}}
```

⚠️ 注意：这只在**正面**模板上写条件代码时才有效；如果你写在背面，最后会得到一张正面正常、背面空白的卡片。同样，由于这个机制是检查"正面字段是否为空"，务必把**整个**正面包在条件替换里。例如，下面这种写法就**不会**按预期工作：

```
{{#Expression}}
 {{Expression}}
{{/Expression}}
{{Notes}}
```

## 空白背面

卡片生成**只看卡片的正面**。例如，假设正面模板是：

```
{{Field 1}}
```

背面模板是：

```
{{Field 2}}
```

只要 Field 1 非空，就会生成一张卡片。**如果 Field 2 为空，卡片仍会生成**，最后你会得到一张空白背面的卡片。

如果想避免空白背面，需要在正面模板里用条件把"必需的字段"包起来，例如：

```
{{#Field 2}}
 {{Field 1}}
{{/Field 2}}
```

这样就保证只有 Field 2 和 Field 1 同时非空时，卡片才会被生成。

## 添加空白笔记

当你在 Anki 中添加一条新笔记时，如果卡片模板和笔记字段组合后**没有生成任何卡片**，Anki 会**用第一个模板创建一张空白卡片**。这允许你添加尚未填完的内容，之后再补全字段或调整模板。如果不想保留空笔记，可以用"空卡片"功能删除。

## 完形填空模板

背景信息请参考[完形填空删除](https://docs.ankiweb.net/editing.html#cloze-deletion)章节。

**完形填空笔记类型**的工作方式与常规笔记类型不同。它没有可自定义数量的卡片类型，**只有一个类型**，被一条笔记上所有的完形填空共用。

如上一节"卡片的生成与删除"所述，普通卡片的生成依赖正面有一个或多个非空字段。完形填空笔记类型的生成方式不同：

- Anki 在正面模板中寻找**一个或多个完形填空替换标记**，如 `{{cloze:FieldName}}`。
- 然后在 FieldName 字段中查找**所有完形填空引用**，如 `{{c1::text}}`。
- **每个不同的编号**都会生成一张卡片。

由于完形填空卡片的生成机制不同，`{{cloze:…}}` 标签**不能**用在常规笔记类型上——它们只在完形填空笔记类型中才能正常工作。

条件生成提供了一个**特殊字段**，让你可以判断当前正在渲染的是哪张卡片。例如，想在第一个完形填空上显示"hint1"字段，在第二个完形填空上显示"hint2"字段，可以用如下模板：

```
{{cloze:Text}}

{{#c1}}
 {{Hint1}}
{{/c1}}

{{#c2}}
 {{Hint2}}
{{/c2}}
```
