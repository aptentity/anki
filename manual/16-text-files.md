任何**纯文本**文件，只要字段用逗号、分号或制表符分隔，就可以导入 Anki（需满足以下条件）：

- 文件必须是纯文本（`myfile.txt`）。其他格式如 `.xls`、`.rtf`、`.doc` 必须先另存为纯文本
- 文件必须是 **UTF-8** 编码（见下文）
- Anki 根据文件的**第一行**（非注释行）来确定字段数量。如果后续某些记录字段较少，Anki 会把缺失的字段当作空白处理；如果某些记录字段多了，多余内容不会被导入
- Anki 会自动猜测字段分隔符（逗号、制表符等）。如果猜错了，可以在导入选项窗口中修改分隔符并预览结果。也可以通过添加[文件头](#文件头)来强制指定分隔符

导入时，文本文件中的字段可以映射到笔记的任何字段，包括标签字段。在导入过程中，你可以选择文本文件中的哪个字段对应笔记的哪个字段。

导入文本文件时，你可以选择把卡片放到哪个牌组。⚠️ 注意：如果你的模板设置了 Deck Override（牌组覆盖），卡片会进入覆盖指定的牌组，而不是你在导入时选择的牌组。

**示例文件（三个字段）：**

```
apple;banana;grape
first field;second field;third field
```

## 处理字段中的换行和分隔符

有两种方式：

### 方式一：用引号转义字符

```
hello;"this is
a two line answer"
two;this is a one line field
"this includes a ; (semicolon)";another field
```

引号用于标记字段的起止，所以如果字段内容包含引号本身，需要把**一个双引号替换为两个双引号**来转义：

```
field one;"field two with ""escaped quotes"" inside it"
```

> 💡 用 LibreOffice 等电子表格程序创建 CSV 文件时，会自动处理引号转义。

### 方式二：使用 HTML 换行

```
hello; this is<br>a two line answer
two; this is a one line one
```

需要在导入对话框中勾选 **Allow HTML in fields**（允许字段中的 HTML）才能使 HTML 换行生效。

> ⚠️ 如果使用跨多行的完形填空删除，转义多行**不会**正常工作。这种情况下请使用 HTML 换行。

你还可以在另一个字段中包含标签，然后在导入对话框中把它选为标签字段：

```
first field;second field;tags
```

**注释行**（以 `#` 开头）会被忽略：

```
# this is a comment and is ignored
foo bar;bar baz;baz quux
field1;field2;field3
```

## 电子表格和 UTF-8

如果文件中有非拉丁字符（如重音字母、日语等），Anki 要求文件以 **UTF-8 编码**保存。最简单的方法是用免费的 LibreOffice 而不是 Excel 来编辑文件——它支持 UTF-8 且能正确导出多行内容（Excel 不行）。如果坚持用 Excel，请参见[这份文档](https://docs.google.com/document/d/12YE_FS6A9ANLTESJNtPP116ti4nNmCBghyoJBRtno_k/edit?usp=sharing)。

用 LibreOffice 导出为 Anki 可读文件：点击 **File → Save As**，选择 CSV 作为文件类型。接受默认选项后保存，然后就可以导入到 Anki 了。

## HTML

Anki 可以把从文本文件导入的文本当作 HTML（即网页语言）处理。这意味着带粗体、斜体等格式的文本可以导出到文本文件后再导入回来。导入时勾选 **Allow HTML in fields** 即可。

如果想在文件中使用 HTML 格式，同时包含尖括号或 & 符号，需要做以下替换：

| 字符 | 替换为 |
|---|---|
| `<` | `&lt;` |
| `>` | `&gt;` |
| `&` | `&amp;` |

## 导入媒体

如果想在文本文件导入时包含音频和图片：

1. 把媒体文件复制到[卡片集媒体文件夹](https://docs.ankiweb.net/files.html#user-data)。**⚠️ 不要在媒体文件夹中放子文件夹，否则部分功能无法正常工作。**
2. 在文本文件的某个字段中这样写：

```
<img src="myimage.jpg">
```

或：

```
[sound:myaudio.mp3]
```

导入时记得勾选 **Allow HTML**。

或者你也可以用浏览器的[查找和替换](https://docs.ankiweb.net/browsing.html#find-and-replace)功能批量更新：如果每个字段里都有"myaudio"，想让它播放声音，可以搜索 `(.*)` 替换为 `[sound:\1.mp3]`，同时启用**正则表达式**选项。

> ⚠️ 不要在模板里这样写：
> ```html
> <img src="{{field name}}">
> ```
> Anki 不支持这种方式，原因有两个：一是每张卡片都要渲染一遍，搜索已使用的媒体**会很慢**；二是这个功能对共享牌组用户来说不直观。请用查找和替换代替。

---

## 批量媒体

另一个一次性导入大量媒体的方法是使用[媒体导入插件](https://ankiweb.net/shared/info/129299120)。该插件会自动为指定文件夹中的所有文件创建笔记，正面显示文件名（不含扩展名，所以 `apple.jpg` 的正面会显示"apple"），背面显示图片或音频。如果你想用不同的方式排列媒体和文件名，之后可以[更改创建的卡片的笔记类型](https://docs.ankiweb.net/browsing.html#editing-area)。

## 重复与更新

导入文本文件时，Anki 用**第一个字段**来判断笔记是否唯一。默认情况下：如果导入文件中的第一个字段与卡片集中已有笔记的第一个字段相同，且该笔记类型与你导入的笔记类型相同，则会**更新**现有笔记的其他字段。导入界面中的下拉框可以修改这个行为——也可以完全忽略重复，或把重复作为新笔记导入。

`match scope`（匹配范围）设置控制如何识别重复：

- **note type（笔记类型）**：如果另一个具有相同笔记类型的笔记的第一个字段相同，则识别为重复
- **note type and deck（笔记类型和牌组）**：只有当现有笔记同时也在你导入的目标牌组中时，才标记为重复

如果开启了更新模式，导入的笔记如果已在卡片集中有旧版本，会在原位置（当前所在牌组）**就地更新**，而不是移到你在导入对话框中设置的牌组。如果笔记就地更新，其所有卡片的现有调度信息会被保留。

> 关于 `.apkg` 文件中重复的处理方式，请参见[牌组包](https://docs.ankiweb.net/exporting.html#packaged-decks)章节。

## 文件头

> 需要 Anki 2.1.54+。

Anki 支持在文本文件顶部添加特定的文件头，格式为 `#key:value`，每行一个。

| Key | 可用值 | 作用 |
|---|---|---|
| `separator` | `Comma`、`Semicolon`、`Tab`、`Space`、`Pipe`、`Colon` 或对应的字面字符 | 指定字段分隔符 |
| `html` | `true`、`false` | 指定文件是否被当作 HTML 处理 |
| `tags` | 用空格分隔的标签列表 | 为每条导入的笔记添加标签 |
| `columns` | 用分隔符分隔的字段名列表 | 指定列数，并让导入预览用你提供的名字作为各列的标题 |
| `notetype` | 笔记类型名称或 ID | 预设笔记类型（如果存在） |
| `deck` | 牌组名称或 ID | 预设牌组（如果存在） |
| `notetype column` | `1`、`2`、`3`、… | 指定哪一列包含每条笔记的笔记类型名称或 ID（见下） |
| `deck column` | `1`、`2`、`3`、… | 指定哪一列包含每条笔记的牌组名称或 ID（见下） |
| `tags column` | `1`、`2`、`3`、… | 指定哪一列包含笔记的标签 |
| `guid column` | `1`、`2`、`3`、… | 指定哪一列包含笔记的 GUID（见下） |

### 笔记类型列

通常，一个文件中的所有笔记都会映射到单一的笔记类型。但如果文件中有包含笔记类型名称或 ID 的列，情况就会改变。

这允许你导入**不同笔记类型**的笔记，字段会隐式映射：第一常规列对应任何笔记的第一个字段，第二常规列对应第二个字段，以此类推。**常规列**是指不包含牌组、标签、笔记类型或 GUID 等特殊信息的列。

### 牌组列

通常，导入文本文件产生的新卡片会放在你选择的单一牌组中。但如果文件包含牌组列，笔记的卡片会放到指定牌组中。如果该牌组不存在，会自动创建。

### GUID 列

**GUID** 是 **Globally Unique Identifier**（全局唯一标识符）的缩写。创建笔记时，Anki 会给每条笔记分配一个唯一 ID，用于重复检查。如果你导出笔记时包含了 GUID，之后就可以修改笔记内容后重新导入，只要不修改 GUID 字段，就能更新现有笔记。

> ⚠️ GUID 应由 Anki 创建。如果你要创建自己的 ID（如 `MYNOTE0001`），建议把这些 ID 放在**第一个字段**中，而不是赋给 Anki 的内部 GUID。导入时，Anki 可以用第一个字段或 GUID 进行重复检查，所以不需要把 ID 转成 GUID 才能更新笔记。

> ⚠️ 如果一行有非空的 GUID，重复选项对该行**不生效**。如果提供的 GUID 已在卡片集中存在，不会创建重复笔记。
