导出让你把卡片集的一部分保存为文本文件或 Anki 牌组包。点击 **File → Export** 即可导出。

## 文本文件

如果选择"Notes in Plain Text"（纯文本笔记），Anki 会把笔记内容写到一个文本文件中。每个字段用制表符（Tab）分隔。如果编辑生成的文件，且不修改第一个字段，就可以之后再导回 Anki——Anki 会根据你的编辑更新笔记，前提是导回到同一笔记类型。

> 💡 如果需要同时编辑第一个字段，得先修改笔记类型，让第一个字段变成 ID 编号而不是实际文本。你可以使用[添加笔记 ID](https://ankiweb.net/shared/info/8897764)插件来简化这个过程。

导出时文本会携带所有 HTML 格式内嵌其中，这样再导回时格式才能保留。

## 牌组包

"牌组包"由卡片、笔记、笔记类型以及捆绑在一起的音频或图片组成，文件后缀是 `.apkg` 或 `.colpkg`。你可以用牌组包在用户之间传递卡片，或用来备份卡片集的部分内容。

有两种不同的牌组包：

### 卡片集包（.colpkg）

`.colpkg` 是 "Anki Collection Package"（Anki 卡片集包）的缩写。导出的是**整个卡片集**，包含调度信息。可以用来备份卡片集，或复制到另一台设备。

> ℹ️ 旧版 Anki 创建的卡片集包文件名是 `collection.apkg`。

#### 导入时会发生什么？

导入 `.colpkg` 文件会**删除** Anki 中的所有现有卡片，用 `.colpkg` 中的卡片**替换**。这适合在设备间来回复制卡片集。

导入卡片集包时，媒体文件夹中现有的媒体**不会被删除**。要清理未使用的媒体，使用 **Tools → Check Media**。

#### 导出选项

**Include Media（包含媒体）**

- 开启：本地媒体会包含在内
- 关闭：`.colpkg` 不会包含任何音频、图片或其他媒体

**Support older Anki versions（支持旧版 Anki）（速度较慢 / 文件较大）**

- 开启：创建兼容旧版客户端的 legacy 版本
- 关闭：创建现代版本，导入导出更快，媒体文件会被压缩，但旧版 Anki 无法读取

### 牌组包（.apkg）

`.apkg` 是 "Anki Deck Package"（Anki 牌组包）的缩写。用于导出**单个牌组**（及其所有子牌组）。

> 📌 如果文件名是 `collection.apkg`，那就是卡片集包，不是牌组包。其他以 `.apkg` 结尾的文件都是牌组包。

#### 导入时会发生什么？

导入牌组包时，Anki 会把内容**追加**到你的卡片集中，而不是覆盖。

如果牌组包中的某些笔记之前已被导入过，Anki 会保留**修改时间最新**的版本。

**示例：**

你有一个名为 *Biology* 的牌组在用。现在下载了同一牌组的更新版本并导入，Anki 会把所有更新合并到你的卡片集中。

反过来，如果你先在自己的卡片集里做了修改（如编辑卡片），再重新导入未修改的 `.apkg`，Anki 会**保留你的修改**，不会覆盖。

#### 导出选项

**Include Scheduling Information（包含调度信息）**

- 开启：调度信息（如复习历史）会一起导出
- 关闭：Anki 假定你在与他人分享牌组，会移除所有调度信息（包括已标记和记忆蛀虫标签）。这样其他人拿到的是干净的新牌组。

**Include Deck Presets（包含牌组预设）**

- 开启：牌组预设会一起导出

**Include Media（包含媒体）**

- 开启：本地媒体会包含在内
- 关闭：`.apkg` 不会包含任何音频、图片或其他媒体

**Support older Anki versions（支持旧版 Anki）（速度较慢 / 文件较大）**

- 开启：创建兼容旧版客户端的 legacy 版本
- 关闭：创建现代版本，导入导出更快，媒体文件会被压缩，但旧版 Anki 无法读取
