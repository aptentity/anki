## 自动备份

Anki 会自动备份你的卡片数据。备份内容包括卡片文本和调度信息，**但不包含音频或图片文件**。

自动备份有助于从操作失误中恢复，但**不应作为唯一依赖**。因为备份存储在本地设备上，设备坏了或被盗就找不回来了。建议配合[手动备份](#手动-colpkg-备份)一起使用。

### 恢复

从自动备份恢复的步骤：

1. 打开 Anki，从 File 菜单选择 Switch Profile
2. 点击 **"Open Backup"** 按钮
3. 选择要恢复的备份版本

> ⚠️ 恢复备份后，**自该备份创建以来的所有修改都会丢失**。

从备份恢复后，Anki 会**关闭自动同步和自动备份**。确认恢复的备份正确后，关闭并重新打开 Anki 即可恢复正常状态。

### 创建

自动备份会周期性创建。你可以在[偏好设置](https://docs.ankiweb.net/preferences.html)中调整备份间隔，默认是 **30 分钟**。

以下操作会**立即触发一次备份**，即使距离上次备份还没到设定时间：

- 单向同步下载
- 用 **File > Import** 导入 `.colpkg` 文件
- **Tools > Check Database**

备份超过两天后，Anki 会开始清理部分旧备份。你可以设置要保留多少份**每日、每周、每月**备份。

> ℹ️ Anki 2.1.50 及以上版本创建的备份无法导入到旧版 Anki 中。

## 手动 colpkg 备份

### 恢复

用 **File > Import** 可以从手动备份恢复。

### 创建

在 Anki 2.1.50 及以上版本，可以用 **File > Create Backup** 立即触发一次备份。和普通自动备份一样，不包含媒体文件。

要创建**包含音频和图片**的完整备份：

1. 从 File 菜单选择 Export
2. 确保选中 "Anki collection package (.colpkg)"
3. 勾选 **"include media"** 选项

这会生成一个包含所有卡片和媒体文件的 `.colpkg` 文件。建议把这个文件存到安全的地方，比如另一台设备，或 Dropbox / Google Drive 等云存储服务。

## AnkiWeb

将卡片集[同步到 AnkiWeb](https://docs.ankiweb.net/syncing.html)可以防止设备丢失或被盗。要从 AnkiWeb 恢复卡片集，可以在偏好设置中强制执行单向同步，或在新设备上登录后选择"Download"。

## 删除日志

Anki 会把删除的笔记记录到一个名为 `deleted.txt` 的文本文件中，位于你的 profile 文件夹里。这些笔记是纯文本格式，可以用 **File > Import** 重新导入。

历史上 Anki 只支持同时使用一种笔记类型，所以如果你从不同笔记类型中删除了笔记，需要先把文件按笔记类型拆分成多个文件，再分别导入。

不过，在 Anki 版本 **25.09.2** 之后创建的 profile，`deleted.txt` 顶部会有以下文件头（允许直接导入，无需拆分）：

```
#guid column:1
#notetype column:2
```

> 📌 `deleted.txt` 是每个 profile 独有的，**不同用户 profile 之间不能互换使用**。
