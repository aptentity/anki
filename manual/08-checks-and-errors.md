当你在 Anki 2.1.45+ 中**保存笔记类型修改**或**导出牌组**时，Anki 会检查一些常见错误。这些错误会在以后复习时带来问题，所以 Anki 不会让你在修复之前继续操作。

## 基础

请先阅读[关键概念](https://docs.ankiweb.net/getting-started.html#key-concepts)。

下面的大多数错误都需要你修改笔记类型 / 卡片模板。修改步骤：

- 打开**浏览**界面，查看左侧列表。
- 找到错误信息中提到的笔记类型。必要时用左上角的搜索框。
- 点击该笔记类型，右侧会显示其卡片 / 笔记。
- 点击编辑区顶部的 **Cards…** 按钮，打开[模板界面](https://docs.ankiweb.net/templates/intro.html#the-templates-screen)。

## 具体问题

### 模板语法错误

这类错误表示对[字段替换](https://docs.ankiweb.net/templates/fields.html)语法的使用不正确。

可以通过打开卡片模板界面来修正错误：

- 桌面版：编辑出问题的卡片，然后点击 **Cards…** 按钮
- AnkiMobile：在学习界面查看出问题的卡片时，点击齿轮 → Card Template（卡片模板）

当你修正一个错误时，**该类型的所有卡片都会更新**——你不需要为每张使用该模板的卡片重复同样的修改。

具体要改什么，取决于你收到的错误信息。

**Found '{{Field}}', but there is no field called 'Field'**

模板里引用了一个**不存在的字段**。要修复，请在卡片模板里找到 `{{Field}}`，并删除它。

**Missing }} in {{Field**

模板里出现了 `{{` 但没有匹配的 `}}`。例如：

```
{{Field
```

需要改为：

```
{{Field}}
```

**Missing {{/Field}}**

模板里有 `{{#Field}}` 或 `{{^Field}}`，但没有匹配的 `{{/Field}}`。把 `{{#Field}}` 或 `{{^Field}}` 从模板中删除即可。

**Found {{/One}}, but expected {{/Two}}**

条件替换必须**按打开的相反顺序闭合**。例如下面的模板是错误的：

```
{{#One}}
 {{#Two}}
 {{Three}}
 {{/One}}
{{/Two}}
```

应改为：

```
{{#One}}
 {{#Two}}
 {{Three}}
 {{/Two}}
{{/One}}
```

**Found {{/Field}}, but missing '{{#Field}}' or '{{^Field}}'**

闭合标签必须有对应的开始标签。例如，下面这种写法无效，因为前面没有 `{{#Two}}` 或 `{{^Two}}`：

```
 {{Field}}
{{/Two}}
```

删除闭合标签即可：

```
{{Field}}
```

### 相同的正面

你已经把 Anki 配置成对每条输入都生成**两张相同的问题**。这通常发生在添加新卡片类型时**没做任何修改**就保存了。相同的卡片会让你的复习量翻倍，也会让 Anki 的调度算法效果变差。

要修复，请打开[模板界面](https://docs.ankiweb.net/templates/intro.html#the-templates-screen)，在顶部选中其中一张重复的卡片，然后**用右上角的按钮移除该卡片类型**。这会一并删除使用该卡片类型的所有重复卡片 / 笔记。

### 卡片正面为空

Anki 通过把你填写的字段和模板（模板决定哪些字段出现在正面和背面）结合来显示卡片。如果你收到"卡片正面为空"的消息，意味着：

- 要么正面模板里包含的字段**都没有任何文本**
- 要么你有内容的字段**没有出现在正面模板里**

修复方法：在桌面版上编辑该卡片，点击 **Cards…**，检查至少有一个有内容的字段被包含在正面模板中。可以用 **Add Field** 按钮添加额外的字段。

如果你用的是完形填空（Cloze）笔记类型，请确保你在 Text 字段中包含了**一个或多个**完形填空删除，例如 `{{c1::某个被遮蔽的文本}}`。

如果你使用了"输入答案"功能，请确保在正面也包含了**另一个字段**。

### 完形填空笔记类型缺少 Cloze 过滤器

完形填空笔记类型的正面和背面模板应该都有一个 [cloze 过滤器](https://docs.ankiweb.net/editing.html#cloze-deletion)。如果缺少，需要加回去，Anki 才能正确创建完形填空卡片。

#### 单张空卡片

创建完形填空时，每个不同的编号会变成一张独立的卡片。例如下面的文本会创建 3 张卡片：

```
{{c1::This}} is a {{c2::sample}} {{c3::sentence}}.
```

如果你之后编辑这段文本，删除或修改了某个完形填空编号，对应的已生成卡片可能会变空。例如：

```
{{c1::This}} is a {{c2::sample}}
```

或

```
{{c1::This}} is a {{c2::sample}} {{c1::sentence}}.
```

这两种改动都会让第 3 张卡片变空。查看第 3 张卡片时，你会看到一条消息说"该卡片为空"，可以用"空卡片"功能清理。

操作路径：桌面版主窗口 → **工具** 菜单中的空卡片功能。

⚠️ 使用前请先**检查报告的空卡片**，如有疑问，先用 **File → Export** 导出一份备份。

#### 所有完形填空卡片都为空

如果你不小心修改了卡片模板，可能会导致所有完形填空删除都不显示。如果发生这种情况，请编辑一张出问题的卡片，**记下第一个字段的名称**（通常叫"Text"）。然后：

- 点击 **Cards…** 按钮
- 把正面文本替换为：

```
{{cloze:Text}}
```

- 把背面文本也替换为同样的内容

如果你的字段不叫 Text，把上面的 Text 替换成实际的字段名。
