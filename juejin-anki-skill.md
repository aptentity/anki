# 我在 skillhub 挖到一个 AI Agent Skill，专治"AI 不会用 Anki"

> 一套给 LLM Agent 用的 Anki 方法论，让你的 AI 助手秒变 Anki 学习顾问

## 前言

AI Agent 时代来了，越来越多的人在用 AI 帮自己干活。

但在 Anki 这件事上，AI 表现得**很糟糕**：

- 🤖 **百度百科式卡片**：`Q: Python 是什么？A: 一种编程语言` —— 这不是卡片，是把整段维基百科塞进去
- 🤖 **一卡十事实**：把一个章节所有要点塞进一张卡片，复习 3 次就放弃
- 🤖 **瞎建结构**：不问你的需求就建好牌组树，一周后无法维护
- 🤖 **不会诊断**：你说"复习好累"，它回"加油"，完全不会看你的统计找问题

**为什么会这样？—— 因为 AI 缺少 Anki 方法论**。

Anki 是个**入门简单、精通极难**的工具，背后涉及认知科学、记忆曲线、间隔算法。最近我在 **skillhub** 上发现了一个沉淀得非常好的 **AI Agent Skill**，把 Anki 的使用方法论系统地组织了起来。

它不是 Anki 插件，**而是一套结构化 Prompt**。把 SKILL.md 放进你的项目，AI Agent 在任何对话场景下，都能像专业 Anki 顾问一样帮你：

- 📝 拆解素材为原子事实
- 🎯 选择最合适的卡片格式
- 🚨 检测"病卡"并给出重写建议
- 📊 分析统计、找出"难度地狱"
- 🗂️ 优化牌组结构和标签策略

**本文 3500 字**，会拆解这个 Skill 的设计思路和核心内容。看完后，你也可以用同样的方法为其他领域写 Skill 👇

> 如果你也在用 AI Agent 辅助学习，**点赞 👍、收藏 ⭐** 是对我最大的支持！

## 一、这个 Skill 长什么样？

整个 Skill 是一个标准化的目录：

```
anki/
├── SKILL.md           # 入口，定义 Agent 的角色和工作流
├── cards.md           # 卡片设计与格式
├── retention.md       # 记忆保持优化
├── organization.md    # 牌组组织策略
└── language.md        # 语言学习专题
```

**5 个文件，分工明确**：

| 文件 | 解决的问题 | 适合谁读 |
|------|-----------|---------|
| `SKILL.md` | Agent 怎么工作、5 条铁律 | 所有 Agent 调用者 |
| `cards.md` | 单卡怎么写、4 种格式选择 | 创建/重写卡片时 |
| `retention.md` | 保持率优化、FSRS 算法 | 复习痛苦时 |
| `organization.md` | 牌组结构、标签、合并 | 牌组混乱时 |
| `language.md` | 词汇卡、假朋友、频率 | 学语言时 |

**工作流程**：当用户提出 Anki 相关需求，Agent 先读 `SKILL.md` 确认自己角色，再按需加载 `cards.md`/`retention.md` 等专项文件。这叫**按需上下文加载**——既节省 token，又让方法论可复用。

## 二、SKILL.md：给 Agent 的"人设"

`SKILL.md` 是 Agent 的入口。**关键设计有 3 处**：

### 1. 5 条铁律（Critical Rules）

```markdown
1. One fact per card — Never cram multiple concepts
2. Answers must be unambiguous
3. Context is mandatory
4. Cloze ≠ always better
5. Leech threshold matters
```

为什么是"铁律"而不是"建议"？因为**这些是反例的根源**。AI 的本能是"看起来完整"，但卡片要的是"原子化"。明确写入"铁律"后，Agent 在任何场景下都会先检查自己是否违反。

### 2. 红色预警清单（Red Flags）

列出了 5 个 Agent 必须能识别的"病卡信号"：

- 含多个填空测不同事实
- 能用排除法答对的题
- "What is X?" 没指定领域
- 列表伪装成单卡
- 反向卡无意义

**这个清单是关键**。把"应该避免"具象化成"看到 X 就是错的"，Agent 才能稳定执行。

### 3. 能力边界（Scope）

```markdown
This skill ONLY: 创建用户明确提供的素材的卡片
This skill NEVER: 未经请求读文件、自动访问 Anki 数据
```

**边界声明**比"能做啥"更重要。Agent 知道自己**不能做什么**，才不会越权或乱猜。

## 三、cards.md：单卡设计的方法论

这是整个 Skill 最核心的文件，分 4 个模块。

### 3.1 原子化原则

**每张卡片只测一个事实**。判断标准：

| 自检问题 | 通过条件 |
|---------|---------|
| 这张卡能只有一个正确答案吗？ | ✅ |
| 只看问题能知道答案唯一吗？ | ✅ |
| 我在测试回忆还是识别？ | ✅ 是回忆 |

❌ 反例：`X 的症状有哪些？`（答案是列表）
✅ 正例：`X 引起 [症状]，是什么疾病？`（单一事实）

### 3.2 4 种卡片格式

不同场景选不同格式：

| 格式 | 最佳场景 | 常见陷阱 |
|------|---------|---------|
| **Q&A** | 概念、解释、流程 | 问题太宽泛、答案出现在问题中 |
| **填空 (Cloze)** | 定义、序列、具体数字 | 多个填空测不同事、填空太长/太短 |
| **图片遮挡** | 解剖、图示、地图 | 遮挡太多/太少、图片模糊 |
| **双向卡** | 单词（识别+回忆） | 反向无意义（如"巴黎是？的首都"） |

**好填空的特征**：
- 定义中的关键词
- 数字、日期、数量
- 流程中的人名
- 小集合中的一个元素

**坏填空举例**：
- `{{c1::Je m'appelle Marie.}}`（整句填空→没测任何东西）
- `J'{{c1::ai}} {{c2::mangé}}`（多填空测不同事）

### 3.3 上下文线索

**每张卡片需要 3 类信息**：

1. **领域标识**：牌组名、标签或文本前缀（如 `[Cardiology]`）
2. **区分性信息**：和相似事实区别的关键
3. **来源引用**（可选）：方便后期核对

❌ 反例：`What does ACE stand for?`（医学？扑克牌？ACE 公司？）
✅ 正例：`[Cardiology] 在血压调节中，ACE 是什么的缩写？`

### 3.4 批量生成工作流

Agent 拿到素材时按这个流程处理：

```
1. 抽取事实 → 2. 查重 → 3. 选格式 → 4. 加上下文 → 5. 加标签 → 6. 自检
```

**第 1 步是核心**：让 AI 列出"哪些是独立可测的事实点"，比"让 AI 写卡片"效果好 10 倍。

## 四、retention.md：记忆保持的科学

这个文件解决"复习好累但效果差"的问题。

### 4.1 目标指标

| 指标 | 目标 | ⚠️ 警戒线 |
|------|------|-----------|
| 保持率 | 85-90% | <80% 或 >95% |
| 平均难度系数 | 250% | <200%（难度地狱） |
| 水蛭占比 | <2% | >5% |
| 每日复习 | 可持续 | >300/天（崩溃风险） |

### 4.2 难度地狱（Ease Hell）

**症状**：很多卡片难度系数 <200%，每天反复出现，复习时间爆炸。

**形成机制**（恶性循环）：

```
失败 → 难度-20% → 间隔缩短 → 又失败 → 难度再降 → ...
```

**3 步修复**：

1. **手动重置**：受影响卡片难度设为 250%
2. **切换 FSRS**：基于机器学习的新算法，无难度系数
3. **重写卡片**：90% 的水蛭是卡片设计问题，不是记忆问题

### 4.3 FSRS vs SM-2

| 算法 | 特点 | 推荐度 |
|------|------|-------|
| SM-2 | Anki 默认，固定算法，难度系数可能失控 | ⭐⭐ |
| FSRS | 机器学习，无难度系数，自动调优 | ⭐⭐⭐⭐⭐ |

**什么时候切换 FSRS**：
- ✅ 处于难度地狱 → 立即切
- ✅ 复习数 >1000 → 数据充足
- ❌ 复习数 <1000 → 数据不足，不要切

### 4.4 复习积压恢复

假期后回来看到 1000+ 待复习？**不要恐慌式复习**：

1. 设置每日上限 100-150 张
2. 多出来的隐藏（Bury）
3. 新卡数设为 0
4. 用筛选牌组优先复习最紧急的

## 五、organization.md：牌组组织哲学

### 5.1 扁平 vs 深层

**坑**：很多人喜欢建 `A::B::C::D` 的深层结构。

**真相**：深层结构难导航、不灵活。**推荐**：
- 1-2 层牌组（按大分类）
- 标签用于具体主题、来源、难度

```
Medical
├── Preclinical
├── Clinical
└── Boards
```
配合标签：`#cardiology` `#high-yield` `#source::FirstAid`

### 5.2 标签策略

**层级标签**用 `::`：
- `anatomy::upper_limb::muscles`
- `pathology::cardio::arrhythmias`

**功能标签**：
- `#leech` — 已知问题卡
- `#rewrite` — 待改进
- `#high-yield` — 考试重点

> 💡 **不要过度打标签**。从来不按某个标签筛选的话，删掉它。

### 5.3 何时拆分牌组

| 该拆 | 不该拆 |
|------|--------|
| >2000 张且主题不同 | 仅为了"组织好看" |
| 需要不同的学习节奏 | 主题高度交叉 |
| 需要不同的设置 | 卡片数 <500 |

### 5.4 维护节奏

| 频率 | 任务 |
|------|------|
| 每周 | 清理暂停卡、处理水蛭、看复习预测 |
| 每月 | 审计标签、查重复、看 6+ 月未用的卡 |
| 考试前 | 创建筛选牌组、调整优先级 |

## 六、language.md：语言学习专题

学语言是 Anki 最常见的场景，单独开一个文件。

### 6.1 词汇卡模板

**最低字段**：单词、释义、例句、音频
**推荐添加**：音标、词性、性别、频率、搭配

### 6.2 单词卡 vs 句子卡

| | 单词卡 | 句子卡 |
|--|--------|--------|
| ✅ 优点 | 创建快、识别强 | 真实用法、测试理解 |
| ❌ 缺点 | 无上下文、干扰多 | 创建慢、可能靠上下文猜 |

**最佳实践**：单词+例句开始，难辨词再补句子卡。

### 6.3 假朋友警示卡

特别为易混淆词创建对比卡：

```
Front: [ES→EN] "Embarazada" 是什么意思？
Back: 怀孕（不是 embarrassed，那是 "avergonzada"）
```

### 6.4 频率优先

按语言学家统计：

1. **前 1000 词** — 覆盖约 85% 书面语 / 75-80% 口语
2. **前 3000 词** — 覆盖约 95% 日常内容
3. **领域专业词** — 按需

按频率打标签：`#A1` `#A2` `#B1` `#B2` `#C1`

## 七、Agent 调用范式

**这个 Skill 怎么用？**

在支持 Skills 的 IDE（Trae、Cursor 等）中，把这个目录放到 `.trae/skills/anki/`。之后你跟 AI 说：

```
"帮我把这段医学笔记做成 Anki 卡片"
"分析我的统计，找出难度地狱"  
"重写这张总是失败的卡片"
"整理我的英语牌组，按频率分组"
```

Agent 会自动加载 Skill 知识，**按 5 条铁律 + 4 种格式**生成卡片。**这才是真正的"AI Agent"**——不是问啥答啥的聊天机器人，而是按方法论稳定输出的助手。

### 对比其他 Agent 设计

| 设计方式 | 缺点 |
|---------|------|
| ❌ 把方法论塞进 System Prompt | Token 浪费、不能复用 |
| ❌ 让 AI "自由发挥" | 质量不稳定 |
| ✅ **Skill 文件按需加载** | 节省 token、可复用、可调试 |

过去我们写 Wiki、写文档、写代码注释，是给**人**看的。

**Skill 与传统文档的核心区别**：
- 给人看的：要可读、有故事、有情感
- 给 AI 看的：要**结构化、原子化、有边界**

**这个 Skill 的价值**：

- ✅ 沉淀了领域专家的 Anki 方法论
- ✅ 让 AI 输出稳定可靠
- ✅ 易于版本管理和迭代
- ✅ 跨项目、跨 IDE 复用
